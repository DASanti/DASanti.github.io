Automated burr detection and aluminum profile classification cell

Worked on the development of a prototype cell that uses image recognition, machine learning, and a robotic arm to handle, analyse, and classify aluminium profiles by detecting the presence of burrs around drilled holes. Image acquisition and pre-processing was carried out using python (OpenCV) and image recognition was implemented using a machine learning algorithm set-up in Orange. Since access to the robotic arm was limited, a simulation of the physical process was developed using Robot Studio. This whole project was achieved closely following an AGM (Agile Project Management) approach and was graded according to its principles, as well as other mechatronic design fundamentals.

Micro CNC lathe: CAD/CAM design and parameter research

Designed the CAD model of a CNC lathe in SOLIDWORKS and aided in the calculation of the parameters required to make it capable of turning an aluminium billet into a chess piece. Digital vehicle dashboard prototype for driver cabin using CAN bus

Developed a circuit that simulates communication between a vehicle and the system through a real CAN network and aided in the design of an Android Automotive OS based interface capable of displaying received information on a basic GUI. The circuit was built using an Arduino UNO running a simple program to emulate a vehicle ECU sending messages, two CAN controllers, and a Raspberry Pi functioning as the brain of the operation. The graphic user interface was developed in Android Studio as an android application. Communication between simulated ECU and RPi was achieved, and custom messages could be received and decoded correctly.

Automatic Rubik's cube solver: CAD model and software

Developed a robot capable of solving a Rubik's cube. The CAD design of the robot was built using SOLIDWORKS and the image processing algorithm was created in python. Using a webcam, OpenCV, and other libraries, images of each of the faces of the cube were captured and analysed. The algorithm was programmed to create arrays containing the color combinations of each face. These were then fed into a Kociemba algorithm, which returns a string with the necessary steps to solve the cube. Each step was then interpreted by a program in an Arduino Uno, which in turn sent to the step-motor drivers the required signals to twist and solve the cube. Due to pandemic limitations, a physical prototype could not be built, but all developments on this project were graded on the basis of an actual and eventual physical implementation.

Robotic Hand and 3D sensor to mimick user's hand movements

Built an articulated robotic hand that mimics the movement of a user's hand using a precision 3D (Leap Motion sensor). The sensor could be placed in front of the user, detecting each movement of the user's hands, and generating information in real time about the position of each joint. This information was fed into a control system (Python and ESP8266) which manipulated the motors of the robotic arm. The prototype could copy and display basic signs, moving each finger individually and forming intermediate positions depending on the actual position of the user's fingers.

PLC controlled elevator programmed with SIMATIC STEP 7

Programmed a simulated PLC running in Siemen's S7-PLCSIM using TIA Portal to run an also simulated multiple-floor elevator with several functions. The elevator program could perform the most common functions of a real-life elevator, such as attending several floors in an efficient manner, display the current floor of the elevator, slow down the cabin before arriving to the requested floor, prevent motor overexertion at the ends of the shaft and by warning users of weight overload, manage all indicators such as requested floor button light and floor up/down button light, prevent door closing when a user is in the way, and an emergency button that halts all functions and runs a specified routine.
